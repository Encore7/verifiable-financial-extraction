# new-migration — Alembic migrations

> **Status: LIVE (as of M1).** Async env at `alembic/env.py` (URL from
> `config.get_settings()`, `target_metadata = db.models.Base.metadata`). Models
> live under `src/db/models/`; the initial schema migration is in
> `alembic/versions/`.

## Workflow

```bash
make infra-up                       # Postgres must be running to autogenerate
make revision m="add foo column"    # autogenerate from model changes
# review the generated file in alembic/versions/ — then:
make migrate                        # alembic upgrade head
uv run alembic check                # assert no model<->migration drift
```

## Rules

- Define/modify SQLAlchemy 2 (typed `Mapped`) models under `src/db/models/`, and
  **register new modules in `src/db/models/__init__.py`** — autogenerate only
  sees what's imported onto `Base.metadata`.
- Constraint/index names come from the naming convention in `src/db/base.py`;
  keep it so names stay deterministic across migrations.
- **Always review the autogenerated SQL** — it misses some changes (server
  defaults, type/enum edits, some index changes). Confirm the down-revision and
  that `downgrade()` reverses cleanly.
- `make revision` requires a running DB (it diffs against live Postgres). The
  migration must be reproducible against the `pgvector/pgvector:pg16` image.
- Autogenerated files under `alembic/versions/` are excluded from ruff.
- LangGraph checkpoint tables are managed by AsyncPostgresSaver, not here.
